This section lists practical tools for day-to-day mod development in the GregFramework ecosystem.

## By Architecture Layer

- **Core SDK layer (`gregCore`)**: event hooks, bridges, registries, runtime contracts.
- **Plugin/Extension layer**: custom feature stacks (Asset Exporter, Multiplayer, Sysadmin, UI Bridge, etc.).
- **Mod layer**: gameplay mods and standalone MelonLoader modules.
- **Platform layer**: publishing, distribution, docs, and operations tools.

## Practical recommendation

Start with **Core SDK tools** first, then add reverse-engineering and deployment tools as required.
