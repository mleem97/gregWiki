# Security Policy

## Reporting Vulnerabilities

Do not open public issues for security vulnerabilities.

**Report securely**: Contact maintainers directly through appropriate channels.

## What Not to Disclose Publicly

- Exploitable vulnerabilities in game code
- Security bypass techniques
- Reverse-engineering methods that violate EULA
- Private API abuse vectors

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark:|
| < 1.0  | :x:               |

Only the latest stable release receives security updates.

## Response Expectations

- Acknowledgment within 48 hours (business days)
- Timeline for fix based on severity
- Credit to reporter (optional, request anonymity)

## Scope

This policy covers:
- Framework code and build system
- Documentation
- Build artifacts

Not covered:
- Game client vulnerabilities
- Third-party mod conflicts
- Private servers
